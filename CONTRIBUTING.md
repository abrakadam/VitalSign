# Вклад в VitalSign

Благодарим за интерес к внесению вклада в VitalSign!

## Как внести вклад

### Сообщение о проблемах

Если вы нашли ошибку или у вас есть предложение:

1. Проверьте существующие [Issues](https://github.com/yourusername/VitalSign/issues)
2. Создайте новый Issue с описанием:
   - Заголовок: Краткое описание проблемы
   - Описание: Подробное описание
   - Шаги воспроизведения
   - Ожидаемое поведение
   - Фактическое поведение
   - Скриншоты (если применимо)
   - Окружение: OS, Python версия, версии библиотек

### Pull Requests

1. **Форкните репозиторий**
   ```bash
   https://github.com/yourusername/VitalSign/fork
   ```

2. **Клонируйте ваш форк**
   ```bash
   git clone https://github.com/yourusername/VitalSign.git
   cd VitalSign
   ```

3. **Создайте ветку для фичи**
   ```bash
   git checkout -b feature/AmazingFeature
   ```

4. **Внесите изменения**
   - Следуйте кодстайлу проекта
   - Добавьте комментарии к сложному коду
   - Обновите документацию если нужно

5. **Закоммитьте изменения**
   ```bash
   git add .
   git commit -m 'Add some AmazingFeature'
   ```

6. **Запушьте в ветку**
   ```bash
   git push origin feature/AmazingFeature
   ```

7. **Откройте Pull Request**
   - Опишите ваши изменения
   - Ссылайтесь на соответствующие Issues
   - Убедитесь, что CI проходит

## Руководство по коду

### Структура проекта

- `ui/python/` - GUI на PyQt6
- `main/linux/` - Linux-специфичные анализаторы
- `main/windows/` - Windows-специфичные анализаторы
- `helpers/cpp/` - C++ хелперы
- `helpers/python/` - Python хелперы
- `utils/` - Общие утилиты
- `locales/` - Файлы переводов

### Добавление новой функции

1. Создайте ветку
2. Добавьте функционал в соответствующую папку
3. Добавьте переводы в `locales/ru.json` и `locales/en.json`
4. Обновите документацию
5. Добавьте тесты (если применимо)
6. Откройте Pull Request

### Добавление нового языка

1. Создайте файл `locales/xx.json` (где xx - код языка)
2. Скопируйте все ключи из `ru.json`
3. Переведите значения
4. Добавьте язык в `utils/translator.py`
5. Откройте Pull Request

### Кодстайл Python

- Используйте 4 пробела для отступов
- Максимальная длина строки: 100 символов
- Используйте type hints для функций
- Добавляйте docstrings к функциям и классам
- Следуйте PEP 8

```python
def example_function(param1: str, param2: int) -> bool:
    """
    Описание функции.
    
    Args:
        param1: Описание параметра 1
        param2: Описание параметра 2
    
    Returns:
        bool: Описание возвращаемого значения
    """
    return True
```

### Кодстайл C++

- Используйте 2 пробела для отступов
- Используйте camelCase для переменных
- Используйте PascalCase для классов
- Добавляйте комментарии к сложным частям

```cpp
class ExampleClass {
public:
    int exampleMethod(int param) {
        return param * 2;
    }
};
```

## Тестирование

### Запуск тестов

```bash
# Запуск всех тестов
python -m pytest tests/

# Запуск конкретного теста
python -m pytest tests/test_example.py

# С покрытием кода
python -m pytest --cov=. tests/
```

### Ручное тестирование

1. Запустите GUI: `bash scripts/run_gui.sh`
2. Проверьте все вкладки
3. Проверьте смену языка
4. Проверьте настройки
5. Проверьте на разных ОС если возможно

## Документация

### Обновление README

При добавлении новых функций обновите `README.md`:
- Добавьте описание функции
- Обновите список возможностей
- Добавьте инструкции если нужно

### Обновление INSTALL.md

При изменении процесса установки обновите `INSTALL.md`:
- Добавьте новые способы установки
- Обновите существующие инструкции
- Добавьте решение новых проблем

## Выпуски (Releases)

Процесс создания релиза:

1. Обновите версию в `__init__.py` или соответствующем файле
2. Обновите `CHANGELOG.md`
3. Создайте тег: `git tag v1.0.0`
4. Запушьте тег: `git push origin v1.0.0`
5. Создайте Release на GitHub
6. Добавьте заметки к выпуску
